7  Common Functions

This chapter covers the small set of functions that will carry the majority of your day-to-day work. Each one maps to something you already do in Excel, SAS, or Access — the goal here is naming the R equivalent and giving you a working recipe, not re-teaching the underlying logic you already have.

All examples continue using static_file, loaded back in Your First Import.

7.1 Cleaning messy column names first

Real files rarely arrive with clean, consistent column names. Before doing anything else, it’s worth knowing about janitor::clean_names() — genuinely one of the highest-value, lowest-effort functions in this entire book:

library(janitor)

static_file_new_names <- static_file |> clean_names()

names(static_file_new_names)[1:10]
 [1] "static_file_data_id" "school_code"         "school_name"        
 [4] "school_ctds"         "district_code"       "district_name"      
 [7] "district_ctds"       "charter"             "county"             
[10] "aoi"

This converts inconsistent capitalization, spaces, and punctuation in column names into a single consistent format (typically snake_case) automatically. Our Static File sample already has clean names, so you won’t see dramatic changes here — but the moment you load a real Excel export with column headers like "School Name " (trailing space included) or "% Proficient", this single line will save you real frustration. Use it by habit on every new file you import.

case_when() handles multiple conditions in sequence — the equivalent of a nested IF() chain, but far easier to read and far harder to make a logic error in, since it’s evaluated top to bottom and stops at the first matching condition:

static_file |>
  mutate(performance_label = case_when(
    Performance == 1 ~ "Minimally Proficient",
    Performance == 2 ~ "Partially Proficient",
    Performance == 3 ~ "Proficient",
    Performance == 4 ~ "Highly Proficient",
    TRUE              ~ NA_character_
  ))

The final TRUE ~ ... line is your catch-all — anything that didn’t match an earlier condition lands here. 7.11 Putting several together

This is where the pipe (|>) starts to earn its keep — chaining several of the functions above into a single readable sequence:

static_file |>
  filter(Subject == "ELA", !is.na(Performance)) |>
  select(SchoolName, DistrictName, Performance) |>
  mutate(performance_label = case_when(
    Performance == 1 ~ "Minimally Proficient",
    Performance == 2 ~ "Partially Proficient",
    Performance == 3 ~ "Proficient",
    Performance == 4 ~ "Highly Proficient"
  )) |>
  arrange(DistrictName)
# A tibble: 7,254 × 4
   SchoolName DistrictName Performance performance_label   
   <chr>      <chr>              <dbl> <chr>               
 1 School 311 District 104           1 Minimally Proficient
 2 School 311 District 104           1 Minimally Proficient
 3 School 311 District 104           1 Minimally Proficient
 4 School 311 District 104           1 Minimally Proficient
 5 School 311 District 104           3 Proficient          
 6 School 311 District 104           1 Minimally Proficient
 7 School 311 District 104           2 Partially Proficient
 8 School 311 District 104           2 Partially Proficient
 9 School 311 District 104           1 Minimally Proficient
10 School 311 District 104           2 Partially Proficient
# ℹ 7,244 more rows

Read top to bottom as one sentence: take the Static File, keep only assessed ELA records, keep just these columns, add a readable label, then sort by district. Each line is a single, understandable step — and the whole sequence reads roughly the way you’d describe the task out loud to a colleague.
