1  Why R First

If you’re going to invest real time learning a programming language for education data work, the honest question is: why R, and not Python — especially when Python is more general-purpose and shows up more often in broader data science and machine learning conversations?

1.1 What Python is genuinely better at

It’s worth being upfront about this rather than pretending the choice is obvious. Python tends to win on:

  • Stability and interoperability — it’s the default language of choice in most general software engineering contexts, and integrates more easily into production systems, web applications, and APIs outside of data analysis.
  • General-purpose use — Python is a better choice if you’re building something that isn’t primarily about analyzing a dataset (an application, a scheduled automation system, a web scraper).
  • Machine learning and AI tooling — the dominant ML/AI libraries (PyTorch, TensorFlow, most LLM tooling) are Python-first, with R support often secondary or absent.

If your work were mostly software engineering, Python would likely be the better starting point. But that’s not most assessment and accountability work.

1.2 What R is genuinely better at, for this work specifically

  • It works out of the box for data analysis. Reading a file, summarizing it, and making a publication-quality chart takes fewer lines and less setup friction in R than in Python, where you’re more often assembling several libraries yourself.
  • A more controlled, uniform ecosystem. The “tidyverse” family of packages (which you’ll use constantly in this book) was designed by a small, coordinated team to work together consistently — function names, argument order, and output formats follow shared conventions. Python’s data ecosystem is powerful but more fragmented, with more ways to do the same thing.
  • Visualization and presentation. R’s ggplot2 (and the broader “grammar of graphics” approach you’ll learn in Visualization) remains one of the most expressive and widely respected charting tools in any language, and R’s document-generation tooling (Quarto, which this entire book is built with) makes producing polished reports and slides remarkably straightforward.

1.3 The most important reason: a shared language with your field

Here’s the argument that matters most for your day-to-day work: there is a growing, active community in education data and accountability that works in R, and that shared language is very valuable.

A few concrete examples:

  • The SGP package — used to calculate Student Growth Percentiles, a core concept in state accountability systems — is written in R. If your state’s growth model uses SGP methodology, you may already work with some R code and data.
  • Researchers are actively building suites of tools that can simplify or perform common tasks. For example, Nathan Dadey’s ReportR package works directly with the same data format used to produce Student Growth Percentile calculations and related calculations. You can import your file, and create useful reports and analytics. Tools built by and for people doing exactly your kind of work are shareable and adaptable in R.
  • When you hit a wall, there may be an analyst who’s solved your exact problem in R, in a forum post, a GitHub repo.
  • Finally, since your analysis will exist fully as text, any part of the process can be copy/pasted to an AI tool; with many education researchers working in R, you can often receive answers very tailored to your needs.

Learning the language your professional community already speaks can reduce the time you spend troubleshooting.

1.4 You’re not choosing a side forever

This isn’t a permanent fork in the road. There’s a growing “polyglot” movement actively working to align R and Python rather than treat them as competitors. Wes McKinney, co-creator of Python’s pandas, and Hadley Wickham, creator of R’s tidyverse — two people who have each individually shaped the modern data science landscape — both work at the same nonprofit, Posit, building tools designed to let the two languages work together and share strengths (Arrow and Parquet, which you’ll encounter in Database Connections, are good examples of this convergence in practice).

Practically, this means: nearly everything you learn here — thinking in data frames, vectorized operations, the logic of filtering and grouping and joining — transfers directly if you ever do pick up Python later. You’re not learning a dead-end skill. You’re learning the dialect your field already speaks, with a clear bridge to learn Python if and when you need it.
